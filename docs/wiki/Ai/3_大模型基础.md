# 🤖 大语言模型 (LLM) 基础笔记

## 一、 大模型学习与进化生命周期
> 大模型并非一出生就会聊天，它经历了一个从“通才”到“专才”的进化过程。

1.  **预训练 (Pre-training)：** 学习海量文本，掌握统计规律。此时模型只是个“超级完形填空高手”。
    * **产出：** 基础模型 (Base Model)。
    * **本质：** 得到一系列**权重矩阵**（将知识编码在参数中）。
2.  **指令微调 (SFT - Supervised Fine-Tuning)：** 通过“问题-答案”对，教会模型听从人类指令，具备对话能力。
3.  **人类反馈强化学习 (RLHF)：** 通过人类排序评分，让模型输出更符合人类价值观（有用、诚实、无害）。

---

## 二、 从 Seq2Seq 到 Transformer 的演进

### 1. 传统 Seq2Seq (基于 RNN/LSTM)
传统的序列到序列模型主要依赖循环神经网络 (RNN)。
* **结构：** 编码器 (Encoder) 将输入压缩成一个固定维度的向量（上下文向量），解码器 (Decoder) 再将其逐个解压输出。
* **缺点：**
    * **长距离依赖遗忘：** 序列越长，前面的信息在传递到后面时损失越严重（梯度消失）。
    * **串行计算低效：** 必须处理完$token_{n}$ 才能处理 $token_{n+1}$，无法利用 GPU 的大规模并行能力。

### 2. Transformer 的革命
Transformer 彻底抛弃了循环结构，完全基于 **注意力机制 (Attention)**。
* **并行化：** 在处理输入序列时，所有 token 同时输入，计算效率呈几何级数提升。
* **全局视野：** 无论两个 token 距离多远，它们在注意力层中的“交互距离”都是恒定的，有效解决了长文本遗忘问题。

## 三、 核心：注意力机制 (Attention)

### 1. 视觉比喻
类似于人看合影时会聚焦于“主角”或“熟人”。在处理文本时，模型会根据当前语境，动态地给序列中不同的 token 分配不同的“注意力权重”。

### 2. QKV 矩阵：搜索与匹配系统
大模型通过权重矩阵将每个 token 映射为三个向量：
* **Query (Q) - “我想找什么”：** 代表当前 token 搜索信息的意图。
* **Key (K) - “我有什么”：** 代表其他 token 包含的信息标签。
* **Value (V) - “我的具体内容”：** 代表 token 携带的实际信息数值。

**计算过程：**
1.  **计算相关性：** 计算 $Q$ 与所有 $K$ 的点积（内积），衡量它们之间的相关性。
2.  **归一化：** 通过 **Softmax** 函数将分值转化为 0~1 之间的权重（Attention Score），所有权重之和为 1。
3.  **加权求和：** 用这些权重对 $V$ 进行加权，得到最终包含上下文信息的输出向量。

> **举例：** `“猫在地上跑，它很可爱”`
> 当模型处理到 `“它”` 时，`“它”` 的 Query 会与 `“猫”` 的 Key 产生极高的匹配分，从而让模型在处理 `“它”` 的语义时融合更多 `“猫”` 的特征。

## 四、 Transformer 的其他关键组件

### 1. Embedding (词嵌入)
将离散的文字转换为高维空间的连续向量，使计算机能理解“猫”和“狗”在语义上的接近程度。

### 2. Positional Encoding (位置编码)
由于 Transformer 同时处理所有 token，它本身不具备位置感（分不清“张三打李四”和“李四打张三”）。
* **对策：** 在 Embedding 中注入位置信息，让模型知道每个单词在句子里的精确顺序。

### 3. Multi-Head Attention (多头注意力)
模型同时运行多个独立的注意力机制（多头）。
* 有的“头”关注**语法结构**（如动宾关系）。
* 有的“头”关注**指代关系**（如代词指向谁）。
* 这让模型能从多个维度同时深刻理解一句话。

## 五、 总结对比

| 维度 | RNN (旧) | Transformer (新) |
| :--- | :--- | :--- |
| **处理方式** | 串行（一个接一个） | 并行（全部一起） |
| **长文本支持** | 较差（容易遗忘开头） | 极强（全局关联） |
| **计算效率** | 低（难以利用 GPU 并行） | 极高（现代大规模训练的基础） |
| **核心机制** | 隐藏状态传递 | 自注意力机制 (Self-Attention) |
