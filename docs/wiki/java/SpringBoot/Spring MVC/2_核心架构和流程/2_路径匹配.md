## URL的三个层次

| 组件名称         | 示例      | 说明                    |
|--------------|---------|-----------------------|
| Context Path | /my-app | 应用的根目录                |
| Servlet Path | /api    | DispatcherSevlet拦截的路径 |
| Path Info    | /test   | 真正交给spring mvc处理的部分   |

```java
ServletRegistrationBean<DispatcherServlet> registrationBean = 
        new ServletRegistrationBean<>(ds, "/api/*");
```

自定义的`DispatcherServlet`拦截的路径为`/api/*`:
对于请求`/api/test`
- 此时Servlet路径是`/api`
- 剥离Servlet路径
- 真正查找的是`/test`

如果想要请求到达`@GetMapping('/api/test')`

真正的请求应该为：`/api/api/test`
