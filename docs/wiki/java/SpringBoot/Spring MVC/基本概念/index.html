基本概念

Spring MVC 核心知识体系

从核心原理到高级扩展的全面掌握路径

架构原理生产实践源码深度
1. 核心架构与流程
DispatcherServlet
整个 MVC 的大脑，前端控制器
HandlerMapping
根据 URL 找到对应的 Controller 方法
HandlerAdapter
适配器模式实现，支持不同类型的处理器执行
ViewResolver
视图解析（前后端分离模式下主要处理静态资源）
核心执行流程
理解从 Request 到 Response 的全生命周期路径
2. 数据绑定与校验
参数接收注解
@RequestParam, @PathVariable, @RequestBody
数据校验 (Validation)
JSR-303/Hibernate Validator 深度集成
Converter & Formatter
实现自定义类型转换逻辑（如日期、枚举）
WebDataBinder
@InitBinder 实现参数预处理及属性编辑器
3. 请求响应处理
HttpMessageConverter
Jackson/Gson 处理 JSON 序列化与反序列化
内容协商 (Content Negotiation)
根据 Accept 头返回不同格式的数据
结果统一封装
通过 ResponseBodyAdvice 实现全局结果包装
静态资源处理
ResourceHandlerRegistry 配置与缓存优化
4. 切面扩展机制
HandlerInterceptor
拦截器实现（Pre/Post/After）
Filter 过滤器
Servlet 级别拦截（与 Interceptor 的本质区别）
ControllerAdvice
全局 Controller 增强，常用于全局配置
跨域处理 (CORS)
@CrossOrigin 与 WebMvcConfigurer 配置
5. 健壮性与异常处理
@ExceptionHandler
方法级/Controller 级异常捕获
全局异常处理器
@RestControllerAdvice 实现统一错误返回
HandlerExceptionResolver
底层异常解析接口自定义扩展
ErrorController
自定义 404/500 等容器级别错误页面
6. 进阶与自动化
WebMvcConfigurer
代码式配置 MVC 的所有行为
异步请求处理
Callable 与 DeferredResult 的应用场景
MockMvc 单元测试
无需启动容器的接口集成测试
执行链源码分析
深度研究 DispatcherServlet.doDispatch 逻辑

学习建议

  • 01.先理解 DispatcherServlet 的分发逻辑，这是理解所有组件协同工作的前提。
  • 02.重点掌握 HttpMessageConverter，这在现代 RESTful 开发中是最高频的自定义点。
  • 03.通过 WebMvcConfigurer 尝试一次全代码配置，替代传统的 XML 或 Spring Boot 默认配置。
  • 04.利用 MockMvc 编写测试用例，强制自己从调用方视角理解接口设计的合理性。
