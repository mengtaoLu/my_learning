Spring Security学习指南

Security Guide
专家建议

不要试图背下所有过滤器。先从需求入手，再通过 Debug 查看过滤链的工作流程。

Spring Security 核心内容

Java 企业级开发中事实上的安全标准与工业级防御框架。

认证 (Authentication)

解决“你是谁”的问题。验证用户的身份凭证，如用户名密码、验证码、OAuth2 令牌等。

授权 (Authorization)

解决“你能做什么”的问题。在认证基础上，根据用户拥有的权限或角色来控制对资源的访问。

攻击防护

内置对 CSRF（跨站请求伪造）、会话固定、点击劫持等常见安全漏洞的防御机制。

为什么选择 Spring Security?

  • 与 Spring 生态（Boot/Cloud）无缝集成，配置极其简单。
  • 提供强大的 Servlet 过滤器抽象，极易通过扩展实现个性化安全逻辑。
  • 作为工业级方案，内置了对 CSRF、Session 固定等多种攻击的主动防御。
