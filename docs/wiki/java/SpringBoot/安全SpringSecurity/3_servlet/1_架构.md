## 系统架构

```mermaid
graph TD
    subgraph Servlet_Container ["Servlet Container (如 Tomcat)"]
        A[HTTP Request] --> B[Standard Filter A]
        B --> C[Standard Filter B]
        C --> D[DelegatingFilterProxy]
        D --> E[Standard Filter C]
        E --> F[DispatcherServlet]
    end

    subgraph Spring_Application_Context ["Spring Application Context (IoC)"]
        D -- "查找 Bean: 'springSecurityFilterChain'" --> G
        G[FilterChainProxy]
        
        subgraph Security_Filter_Chains ["Security Filter Chains"]
            G --> H{URL Matcher}
            H -- "/api/**" --> SC1[SecurityFilterChain 1: JWT/Stateless]
            H -- "/web/**" --> SC2[SecurityFilterChain 2: Form Login/Session]
            H -- "/**" --> SC3[SecurityFilterChain 3: Default]
            
            subgraph Filters_Inside_Chain ["Filters Inside SecurityFilterChain"]
                SC2 --> F1[UsernamePasswordAuthenticationFilter]
                F1 --> F2[DefaultLoginPageGeneratingFilter]
                F2 --> F3[BasicAuthenticationFilter]
                F3 --> F4[AuthorizationFilter]
            end
        end
    end

    F4 -- "认证通过" --> F

    style D fill:#f9f,stroke:#333,stroke-width:2px
    style G fill:#bbf,stroke:#333,stroke-width:4px
    style SC2 fill:#dfd,stroke:#333
```

- `Servlet Filter Chain`: `Servlet`本身有一个过滤器链。
- `DelegatingFilterProxy`: 作为`SpringApplicationContext`与原生`Servelt`过滤器之间的桥梁，可以加载`Bean`
- `SpringSecurityFilterChain`：这个就是`DelegatingFilterProxy`需要加载的`Bean`
- `SecurityFilterChain`: 真正执行过滤的地方，可以加载`Filter`执行过滤
  - `Filter`通常不是`Bean`，否则可能会被调用两次：
    1. `Spring`容器调用1次
    2. `Spring Security`调用1次
